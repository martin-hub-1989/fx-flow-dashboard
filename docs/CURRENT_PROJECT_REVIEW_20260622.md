## P0：必须先修复

### 1. 数据源方向发生偏移

用户要求的是 Wind MCP，但当前映射与验证代码直接调用：

```text
hexin-ifind-ds-edb-mcp
```

并从 `~/.claude/mcp.json` 读取同花顺 iFind 连接信息。文档却继续使用 “Wind EDB” 表述，混淆了数据来源和授权边界。

修正要求：

- 重新确认并实现 Wind MCP 的实际调用路径；
- 未经用户明确批准，不得以 iFind 替代 Wind；
- iFind 已取得的结果只可作为研究记录，不能继续标记为 Wind 验证结果；
- 所有来源字段必须使用真实来源名。

### 2. 映射验证存在大量假阳性

124 条“已验证”中至少 44 条存在以下一种情况：

- 使用从数据比值反推的任意缩放因子，例如 `658230958`、`667317500`；
- 平均差异超过 1%，部分达到 4%–6%；
- 最大差异达到 12% 以上；
- 只有 1 个重叠点；
- 指标语义不一致，例如国开债映射到国债；
- 人民币指标通过静态比例拟合成美元，而不是使用对应月份汇率换算。

任意比例拟合只能证明两条序列量级接近，不能证明它们是同一指标。

修正要求：

- 禁止自动生成任意 `unit_div_<data-derived-factor>`；
- 单位转换只能来自明确元数据和受控转换表；
- 人民币/美元转换必须引用同期汇率序列；
- 指标名称、口径、频率、单位和方向必须同时一致；
- 低重叠、较大偏差或历史修订必须进入待复核状态，不能进入生产更新计划。

### 3. 生产更新函数尚未实现

`fetch_via_wind_mcp()` 当前直接返回 `None`。因此项目还不能从 Wind MCP 更新任何正式数据。

当前的 `config/update_plan.json` 只有 2 条旧记录，而根据最新映射动态计算应有 124 条，两者已经不一致。

### 4. 两点重叠校验没有真正落实

当前实现的问题：

- `validation_dates` 是用日期减两个月推算，不是读取数据库最后两个实际观测日期；
- 季度序列会生成两个相同日期；
- 日度序列用自然日减 5 天，不保证落在交易日；
- 没有重叠点时直接通过；
- 测试明确把“无重叠也通过”当成正确行为。

这与原始 loop 的硬约束相反。没有两个有效重叠点时必须拒绝自动写入。

### 5. 更新写入与修订审计不可靠

- observations 使用 `INSERT OR REPLACE`，会静默覆盖历史值；
- 没有独立保存旧值、新值、修订原因和确认状态；
- 新增/修订是在写入后判断，因此已写入日期都会被计为修订；
- 原始序列更新后没有自动复算依赖的 derived 序列；
- 数据库中留有一个未结束的 `running` update run。

### 6. 当前 HTML 不能运行

对生成 HTML 内联脚本执行语法检查时出现：

```text
SyntaxError: Unexpected string
```

错误来自图表导出按钮的引号拼接。现有 4.3 MB HTML 不是可交付产物。

此外：

- 只有即远期模块有人工设计的图表；
- 其余 8 个模块只是按 series_id 顺序取前 6 个 raw 序列；
- 所有自动图都是单序列折线；
- 折线默认开启面积填充；
- 图表结构没有继承原 Excel 的分析问题。

### 7. 衍生指标层并未完成

- 数据库有 224 条 derived 序列，但只有 88 条由 Python 复算；
- 136 条仍然只是 Excel 缓存值；
- `metric_definitions` 表为空；
- 复杂 VLOOKUP、PERCENTRANK、CORREL 指标仍未迁移；
- 货物贸易 `V/W/X` 标为 TTM，但代码使用移动平均而非滚动求和；
- “97.6% 匹配率”是按数据点汇总，掩盖了某些关键指标整体不正确的问题。

### 8. 元数据没有形成单一事实来源

当前存在多套不一致口径：

- `series_catalog.json`：407 条；
- SQLite：385 条；
- Data Dictionary：147 raw / 242 derived / 18 manual；
- SQLite：159 raw / 224 derived / 2 manual；
- 86 个序列仍叫 `Column_*`；
- 157 个序列单位为空或 unknown。

必须明确 SQLite 元数据表或一个受版本管理的 catalog 为唯一主数据，其他文档自动生成。

## P1：应在正式看板前完成

- 区分月度、日度、季度和年度序列，不得把季度值填充为月度流量。
- 清理未来月份占位零和 trailing zeros。
- 修正“数据截至日”逻辑：按图表核心序列显示，而不是取模块内最大日期。
- 对 signed flow 不使用普通百分比环比作为默认摘要。
- 颜色不应默认使用红涨绿跌表达“好坏”；Flow 正负应以零轴、深浅和直接标签表达。
- 图表配置应外置为 JSON，而不是硬编码在 Python 中。
- 原 Excel 图表保留/合并/删除决定必须进入可审计清单。

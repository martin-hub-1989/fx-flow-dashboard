## 工作表层结构

### 通用布局模式

大多数工作表遵循一致的布局：

```
行 1-3:    衍生指标区（汇总值，通常为公式 =row7-row8）
行 4-5:    列标题 / 子标题
行 6+:     数据区（日期在左，数值在右）
```

- **冻结窗格：** `3.即远期` 在 H18（冻结到第 18 行和 G 列之前）
- **合并区域：** 仅 `3.即远期` 有（Q3:U3 等，用于分组标题）

### 跨表引用

#### 3.货物贸易 → 3.即远期

```excel
# 行 36 示例
=T36+J36*0.75-VLOOKUP(Q36,'3.即远期'!P:AJ,16,FALSE)*0.75
```

- 引用目标：`3.即远期!P:AJ`（16 列范围）
- 查找键：Q36（3.货物贸易的日期）
- VLOOKUP 第 16 列 = AJ 列（期权 Delta 净变动）

#### 3.证券FI → 3.即远期

```excel
# 行 13 示例
=(AM13-AM14)/VLOOKUP(A13,'3.即远期'!AK:AL,2,FALSE)
```

- 引用目标：`3.即远期!AK:AL`（2 列）
- 用于将债券流动规模标准化（除以即远期某个基数）

### 外部链接 (Wind EDB)

**所有 9 个目标工作表**的日期列（A 列）均使用 `=[1]!edb()`：

- `[1]` = Wind EDB 插件的外部工作簿引用
- `edb()` = Wind 经济数据库查询函数
- 语法：`=[1]!edb(指标代码, 开始日期, 结束日期, 选项)`

在 `openpyxl` 中读取时：
- `data_only=False`：显示公式文本 `=[1]!edb()`
- `data_only=True`：显示缓存的数值结果（上次 Excel 保存时的快照）

**重要：** 原始数据列（B 列及以后）在 Excel 中可能也是 EDB 查询结果。它们的公式文本可能已丢失（被缓存值覆盖），只有通过检查 Excel 中的实际公式才能确认。

## 公式复杂度分级

| 复杂度 | 模式 | 示例 | 所在工作表 |
|--------|------|------|-----------|
| 简单 | `=X7-X8` (行间差) | 环比变化 | 所有 |
| 中等 | `=SUM(H1:I1)` (同列汇总) | 合计 | 代客即期 |
| 中等 | `=VLOOKUP(...)` (跨表查找) | 跨表引用 | 货物贸易, 证券FI |
| 复杂 | `=SUMPRODUCT(...)/SUM(...)` (加权平均) | 移动平均 | 即远期 |
| 复杂 | `=(X4-AVERAGE(X4:X40))/STDEV(X4:X40)` (Z-Score) | 标准化 | 贸易商 |
| 复杂 | `=CORREL(X4:X78,Y4:Y78)` (相关性) | 统计分析 | 贸易商, 证券EQ |
| 复杂 | `=IF(...)` (条件逻辑) | 异常标记 | 多处 |
| 非标准 | `=[1]!edb()` (外部插件) | Wind 数据拉取 | 所有 A 列 |

## 已知数据问题

### 1. EDB 缓存陈旧性

当前 Excel 中的数据是截至保存时的 EDB 快照。日期范围和最新值取决于 Excel 的最后刷新时间。

**最新缓存值（从提取中观察）：**
- 3.即远期：2026-04-30
- 3.代客即期：2026-04-30
- 3.涉外收付：2026-04-30
- 3.货物贸易：2026-04-30
- 3.贸易商：2026-04-30
- 3.服务贸易：2026-05-31
- 3.FDI：2026-05-31
- 3.证券EQ：2026-05-19
- 3.证券FI：2026-04-30

### 2. 零值 vs 缺失值

Excel 中 `0` 与空值未明确区分。导入时需注意：
- 未来月份的占位 0 应视为缺失
- 真实的经济数据为 0（罕见但可能）需保留

### 3. 公式与值的一致性

某些衍生列在 Excel 中是公式，但缓存值可能与公式计算结果不一致（如果 EDB 数据已更新但公式未重算）。导入时以缓存值为准（作为历史快照），复算时以公式逻辑为准。

### 4. 单位不透明

Excel 列标题中标注了单位（"当月值"、"累计值"等），但具体量纲（亿美元 vs 亿人民币 vs 百万美元）需通过数值量级和 EDB 元数据推断。

### 5. 3.证券EQ 日度异常

3.证券EQ 的日期范围为 2018-04 至 2026-05（日度），但实际有效数据点需要检查——可能存在非交易日空白行。

## 数据导入策略建议

1. **以行为单位导入：** 每行是一个时间点，A 列是日期
2. **跳过汇总行：** 行 1-5 通常为公式/标题，不应导入为观测值
3. **确定数据起始行：** 各工作表不同，通常行 6-7 开始
4. **处理空行：** 日度数据的非交易日可能为空行
5. **保留公式文本：** 用于血缘文档和后续复算验证

# 原 Excel 图表迁移方案清单

> 源文件：`FX Chartbook - Flow 0515.xlsx`  
> 目标范围：9 个结售汇与 Flow 模块  
> 原图数量：66 张

## 一、总体策略

原 Excel 图表不是全部逐张复刻。迁移原则：

1. 保留原图表达的分析问题；
2. 删除重复图、无标题无定义图和只为 Excel 排版存在的图；
3. 将同一主题的多张季节性图合并为一个带指标选择器的图；
4. 将相同数据的线图/柱图重复展示合并；
5. 主页面只放决策相关图，其他图进入模块 drill-down；
6. 所有图必须绑定 `config/chart_catalog.json`，禁止自动选择前几个序列。

## 二、图表分类

| 分类 | 用途 | 推荐形式 |
|---|---|---|
| 总量趋势 | 看 Flow 方向、拐点和持续性 | 月度柱 + 3MMA/6MMA/12MMA 线 |
| 构成分解 | 看总量由哪些账户或工具贡献 | 堆叠柱 + 总量线 |
| 结汇意愿 | 看结汇/购汇比例和标准化位置 | 双线、Z-score 线、零轴 |
| 季节性 | 看当年相对历史区间 | 月份横轴、历史 min-max band、均值线、当年线 |
| 存量与流量 | 区分余额、变动和累计 | 存量线或流量柱，禁止混在同一轴 |
| 市场关系 | 看 Flow 与汇率、利率、股指关系 | 双轴趋势或散点图 |
| 结构占比 | 看币种、账户或资产类别占比 | 100% 堆叠柱或占比线 |
| 精确查询 | 看最新值和历史明细 | 表格，不强行画图 |

## 三、绘图统一规则

### Chart.js

- 使用 Chart.js + chartjs-plugin-datalabels。
- 最终 HTML 内联依赖，保持单文件离线可用。
- 特殊季节性 band、回归线或浮动区间可使用小型自定义插件。
- 不使用手绘 SVG 作为默认路径。

### 视觉规范

- 背景使用白色或接近白色。
- 主色以深蓝为根色，橙色作为对比，灰色作为历史区间或基准。
- signed flow 默认不用红/绿表达好坏；使用深色/浅色、正负标签和零轴。
- 单序列不显示冗余图例。
- 柱图必须显示零轴。
- 只有单位不同且比较关系明确时使用双轴。
- 线图默认不填充面积。
- datalabels 默认只显示最新点、极值或用户选中的点。
- 标题描述指标，副标题说明单位、频率、时间范围和数据截至日。

### 数据门禁

图表进入正式看板前：

- 所有 series_id 存在；
- 单位明确；
- derived 指标由 Python 复算；
- 最近 24 个有效点校验通过；
- 不含未来占位零；
- 频率一致，或明确使用双轴/频率转换；
- 无 `Column_*` 名称。

## 四、原图逐张处理清单

标记：

- `主保留`：默认展示；
- `次保留`：模块 drill-down；
- `合并`：并入新的统一图表；
- `重做`：分析问题保留，但必须更换绘法或先修数据；
- `删除`：不进入新看板。

## 五、目标看板层级

### 默认展示

每个模块优先展示 2–4 张主图，预计约 24 张。

### Drill-down

分解、关系、季节性和散点图进入展开区，预计约 19 个图表合同或选择器状态。

### 不保留

删除或吸收以下类型：

- 重复的线图/柱图版本；
- 无标题且无法还原指标定义的图；
- 横轴和值范围错位的图；
- 只展示一个已在主图出现指标的单线图；
- Excel 为排版复制的多张同类季节图。

## 七、验收标准

- 66 张原图均有处理决定。
- 新图均有明确分析问题和 takeaway。
- 不存在自动选择 series 的逻辑。
- 主图不超过 4 张/模块。
- 季节性图使用统一组件。
- 所有正式图通过数据门禁。
- 单位、频率、来源和截至日期可见。
- PC 与窄屏均不裁切标题、图例和标签。
- PNG 与 CSV 导出正确。
